## 3. 数据模型

### 3.1 标签组（TagGroup）

```typescript
interface TagGroup {
  id: string;          // 唯一标识（UUID）
  name: string;        // 名称（必填，不可重复）
  color: string;       // 颜色（必选，10 个预设色之一）
  emoji: string;       // emoji（必选，若干预设 emoji 之一）
}
```

**预设颜色（10 个）：**
```
#FF6B6B（珊瑚红）  #FFA94D（柑橘橙）  #FFD43B（阳光黄）
#69DB7C（薄荷绿）  #4DABF7（天空蓝）  #748FFC（薰衣草紫）
#DA77F2（莓果紫）  #F783AC（樱花粉）  #38D9A9（青绿）
#CED4DA（岩石灰）
```

**预设 emoji：**
```
💼 🏠 🏃 🎮 📚 🍔 🎵 ✈️ 💻 🛒 🎨 🏥 🎓 ✍️ 🎂
```

**默认数据：**
- 系统内置一个名为「默认」的标签组（颜色：天空蓝，emoji：💼），不可删除。

---

### 3.2 任务（Task）

```typescript
interface Task {
  id: string;              // 唯一标识（UUID）
  tagGroupId: string;      // 所属标签组 ID（必选，默认「默认」组）
  name: string;            // 任务名称（必填）
  startTime: string;       // 开始时间（ISO 8601，格式 yyyy/MM/dd HH:mm）
  endTime: string;         // 结束时间（ISO 8601，格式 yyyy/MM/dd HH:mm）
  duration: number;        // 任务时长，单位小时（H），由 startTime/endTime 自动计算
  completed: boolean;      // 是否已完成（默认 false）
  createdAt: string;       // 创建时间（ISO 8601）
}
```

**创建规则：**
- `duration` = `(endTime - startTime) / 3600000`，自动计算，用户不可手动修改
- `startTime` 默认为当前时间
- `endTime` 默认为当前时间 + 1 小时
- 时间精度到分钟（yyyy/MM/dd HH:mm）

**完成判定：**
- 用户手动勾选任务格子右上角的圆形勾选框 → `completed = true`
- 每个任务只有一个连续时间段

**默认数据：**
- 系统内置一个「示例」任务，属于「默认」标签组（时间设为当天 10:00-11:00），`completed = false`。

---

### 3.3 数据约束

- **标签组名称全局不可重复**；任务名称允许重复
- **任务允许跨天**：如 23:00 ~ 次日 02:00，格子跨多行显示
- 同一任务的时间段**不允许与其他任务有任何重叠**（开始/结束时间不能有任何交叉，边界相接允许）
- 删除标签组时，其下任务自动迁移到「默认」标签组

## 4. 界面布局

```
┌──────────────────────────────────────────────────────┐
│                    提示消息区域（顶部居中）              │
├──────┬───────────────────────────────────┬────────────┤
│ 左侧栏 │          日程表主区域              │  任务详情   │
│(可隐藏)│     （填满界面剩余空间）           │  （弹出面板）│
│       │                                  │            │
│标签组  │  08  09  10  11  12 ... 22       │            │
│ └默认  │ ┌────────────────────┐           │            │
│ └工作  │ │ 💼 示例任务    [✓] │           │            │
│ └学习  │ └────────────────────┘           │            │
│       │                                  │            │
│已完成  │                                  │            │
│       │                                  │            │
└──────┴───────────────────────────────────┴────────────┘
           ↑
     悬浮功能球（右下角）
```

### 4.1 日程表主区域

- **横轴**：一天内的时间线，默认 08:00 ~ 22:00（可通过滚轮缩放）
- **竖轴**：日期，默认显示从今天开始的 7 天（可往前看过去日期、往后看未来，可通过滚轮缩放）
- **轴标签**：横轴时间标签和竖轴日期标签**始终固定在视野内**（类似 Excel 冻结窗格），标签密度随缩放级别自动增减（如放大时显示更多刻度，缩小时稀疏显示）
- **网格线**：极淡的网格线仅作为视觉参考，本质上**不是单元格**
- 任务以**圆角矩形格子**显示在日程表中，位置对应其日期和时间范围
- 任务格子颜色 = 标签组颜色
- 任务格子内容：`[emoji] 任务名称` + 右上角圆形勾选框
- 完成后格子颜色变灰（降低饱和度和透明度），但不消失

### 4.2 悬浮功能球

- 位于界面右下角，始终悬浮
- 点击展开为 3 个子球：
  - **➕**：新建任务
  - **📍**：定位回今天
  - **🌙/☀️**：切换夜间/日间模式
- 展开和收起带动画

### 4.3 左侧栏（可隐藏）

- **「标签组」区域**：
  - 标题旁有 **+** 按钮（添加标签组）
  - 列出所有标签组，每个标签组旁有 **修改** 按钮（修改颜色、名称、emoji）
  - 每个标签组可展开，显示该标签组下未完成的任务列表
  - 点击任务 → 右侧弹出详情面板
- **「已完成」区域**：
  - 列出所有 `completed = true` 的任务
  - 点击任务 → 右侧弹出详情面板，包含：
    - **「尚未完成」按钮**：将该任务 `completed` 设为 `false`，回到日程表
    - **「删除任务」按钮**：彻底删除，日程表不再显示

### 4.4 任务详情面板（右侧弹出）

- 点击任务格子或侧边栏任务项后从右侧滑入
- 使用 animal-island-ui 的 **Drawer** 组件实现
- 显示并允许修改：标签组、名称、开始时间、结束时间
- 修改时间立即反映到日程表，重叠检查即时生效
- 可关闭面板

## 5. 交互行为

### 5.1 缩放

| 操作 | 效果 | 限制 |
|------|------|------|
| 鼠标滚轮在**横轴**上 | 缩放时间轴（如 8~22 → 6~24） | 最小 4 小时，最大 24 小时 |
| 鼠标滚轮在**竖轴**上 | 缩放日期轴（如 7 天 → 3 天或 14 天） | 最小 1 天，最大 30 天 |
| 鼠标滚轮在**日程区域**上 | 同时缩放时间轴和日期轴 | 同上限制 |
| 缩放起点 | 以鼠标位置为中心缩放 | — |
| 体验要求 | **必须平滑**（CSS transition / requestAnimationFrame） | 不得卡顿 |

### 5.2 视图平移

- **右键按住拖拽**移动整个日程表显示区域
- 必须阻止浏览器默认右键菜单
- 平移时不改变缩放级别
- 体验要求：**必须平滑**

### 5.3 任务格子拖拽

| 操作 | 行为 |
|------|------|
| **整体拖拽** | 按住任务格子拖动 → 整体平移，时长不变，修改开始/结束时间 |
| **拖拽左侧边缘** | 修改开始时间，结束时间不变，时长变化 |
| **拖拽右侧边缘** | 修改结束时间，开始时间不变，时长变化 |
| **拖拽约束** | 必须 snap 到时间网格（最小 15 分钟粒度） |
| **冲突检测** | 放置位置不能与任何其他任务有时间重叠，冲突时自动回弹 |

### 5.4 其他交互

- 点击任务格子右上角勾选框 → 切换完成状态
- 点击「今天定位球」→ 视图跳转到今天，不改变缩放级别
- 点击详情面板外区域 → 关闭面板
- 新建任务 / 修改标签组 → 弹出模态框

## 7. 数据持久化

### 存储方案
- **主存储**：`localStorage`
  - Key：`taketime-data`
  - Value：`{ tagGroups: TagGroup[], tasks: Task[] }` JSON 字符串
- 启动时从 localStorage 读取，不存在则初始化默认数据
- 所有增删改操作后自动保存

### 跨设备方案
- **导出**：按钮 → 将当前数据导出为 `.json` 文件下载
- **导入**：按钮 → 选择 `.json` 文件 → 弹出选项让用户选择：
  - **替换**：清空现有数据，完全用导入数据替代
  - **合并**：保留现有数据，导入文件中的数据；遇到 ID 冲突时由用户选择策略：
    - 跳过重复（保留现有）
    - 覆盖（用导入数据替换）
    - 保留两者（为导入数据生成新 ID）
- 导出文件名格式：`TakeTime_backup_yyyyMMdd_HHmmss.json`

## 10. 关键实现注意事项

1. **animal-island-ui 使用**：npm 安装后务必 `import 'animal-island-ui/style'`，否则组件无样式
2. **右键拖拽**：`preventDefault()` 阻止默认菜单，用 `pointermove` / `pointerup` 实现拖拽
3. **非单元格本质**：日程表背景用 CSS Grid 或绝对定位绘制极淡辅助线，不作为可交互单元
4. **性能**：缩放和拖拽用 `requestAnimationFrame` + CSS `transform`，避免重排
5. **时间冲突检测**：O(n) 遍历已有任务，`startTime < existingEndTime && endTime > existingStartTime` 即为冲突
6. **无第三方拖拽库**：用原生 Pointer Events 实现，减少依赖
7. **响应式**：桌面端优先，移动端作为后续优化
8. **animal-island-ui 组件映射**（结合库的组件）：
   - Button → 功能球、操作按钮
   - Card → 任务格子外壳、详情面板容器
   - Tag → 标签组标签展示
   - Footer → 页面底部信息
   - **Modal** → animal-island-ui 内置，用于新建任务、新建/修改标签组、导入选项
   - **Drawer** → animal-island-ui 内置，用于任务详情面板（右侧滑出）
